# Album Embed Documentation

Albums uploaded to [photos.VSP.ink](https://photos.vsp.ink/) and published to this site live at `/embed/<album-code>` (the album's Chevereto code, e.g. [/embed/fy](/embed/fy)) and are designed to be embedded anywhere an `<iframe>` is allowed — blog posts, an LMS page, or someone else's site. The full list of albums with copy-ready code is at [/embed/](/embed/).

## Basic embed

```html
<iframe src="https://vsp.ink/embed/fy" width="100%" height="800" frameborder="0"
  style="width: 100%; max-width: 100%; height: 800px; border: none;"
  loading="lazy" allowfullscreen title="Album title"></iframe>
```

## Choosing a style

Add `?style=<name>` to the iframe `src` to pick a look. No parameter (or an unknown value) gives the default.

| Style | URL | Suggested iframe height | Best for |
|---|---|---|---|
| `card` | `/embed/fy` *(default)* | 800 | Branded widget with header, description, and carousel |
| `mosaic` | `/embed/fy?style=mosaic` | 800 (scrolls internally) | Seeing the whole album at once as a photo wall |
| `filmstrip` | `/embed/fy?style=filmstrip` | 480 (photos fill any height) | One swipeable row; feels native on phones |
| `editorial` | `/embed/fy?style=editorial` | 500 | When the written story matters as much as the photos |
| `kenburns` | `/embed/fy?style=kenburns` | 600 (fills any height) | Ambient auto-playing slideshow |

The style picker on [/embed/](/embed/) writes these heights into the copied code automatically.

An album can also set its own default with `style: mosaic` (any style name) in the front matter of its file in `_embed/` — the URL parameter still overrides it.

Videos in an album (`.mov`, `.mp4`, `.m4v`, `.webm` URLs, or entries marked `img_video: true`) render as playable video slides/tiles in every style.

## Creating and updating embeds

Embed docs are generated by `~/scripts/create_embed.py`, which reads a **public** album on photos.VSP.ink and writes `_embed/<album-code>.md` into the site folder (`~/Files/vsp-ink`). Alt text for every photo is generated with the `claude` CLI (each image is described by the AI); Chevereto itself has no alt-text field, so alt text lives only in these files.

Albums are identified by their **album code**: the short ID Chevereto puts at the end of the album's URL. For `https://photos.vsp.ink/album/fy` the code is `fy`; for `https://photos.vsp.ink/album/oBJ` it's `oBJ`. Anywhere a command below says `<album-code>`, use that code — or just paste the album's full URL and the script will pull the code out itself.

From any Terminal window, run:

```bash
python3 ~/scripts/create_embed.py                        # list all albums and whether each has an embed
python3 ~/scripts/create_embed.py --new                  # create docs for every album that has none yet
python3 ~/scripts/create_embed.py <album-code>           # create the doc for one album
python3 ~/scripts/create_embed.py <album-code> --update  # re-scrape after adding/removing photos;
                                                         # keeps existing alt text, generates it only for new items
```

Useful flags:

| Flag | Effect |
|---|---|
| `--no-ai` | Skip AI alt text; fall back to the image's Chevereto title/description |
| `--style mosaic` | Set the album's default embed style in the front matter |
| `--force` | Regenerate a doc from scratch (all alt text redone) |
| `--model haiku` | Which claude model writes the alt text (default: haiku) |
| `--delete` | Remove the local doc for an album that's been deleted on photos.VSP.ink |

The usual routine for a new album: upload the photos into an album on photos.VSP.ink, give the album a title and description there, run `python3 ~/scripts/create_embed.py --new`, skim the generated file in `_embed/` (especially the alt text), then commit and push. The album appears at `/embed/<code>` and on [/embed/](/embed/) automatically.

## Removing a deleted album's embed

Deleting an album on photos.VSP.ink doesn't touch its `_embed/<album-code>.md` doc — the file is local to this repo, so the embed page (and any post/page that iframes it) stays up but silently references photos that no longer exist.

Running `python3 ~/scripts/create_embed.py` with no arguments now flags these as **orphaned**: any local `_embed/` doc whose album code no longer resolves on photos.VSP.ink. Clean one up with:

```bash
python3 ~/scripts/create_embed.py <album-code> --delete
```

The script double-checks the album is actually gone (a live 404) before deleting anything — if the album still exists (public, private, or otherwise), it refuses and leaves the doc alone. Note that re-uploading the same photos as a "new" album on Chevereto gets a **new album code**, not the old one — after recreating an album, run `--new` (or create the doc for the specific new code) and update any page/post that embedded the old code to point at the new one.

Notes: the script only sees **public** albums (private, link-only, and password-protected ones are skipped). Videos in an album become playable slides with their poster frame. The album code keeps its exact upper/lowercase in the embed URL — the album at `photos.vsp.ink/album/oBJ` becomes `vsp.ink/embed/oBJ`.
